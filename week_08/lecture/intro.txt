2) Copy-Paste your own code










Why copy-paste your own code is bad?



- Usually you need similar, but still slightly
different code. It is very much likely that you
forget to update it.





Example

const books = [
    { title: 'JavaScript', author: 'D. Ford' },
    { title: 'Clean Code', author: 'R. Martin' },
    { title: 'You Do not Know JS', author: 'K. Simpson' }
];

books.forEach(book => console.log(book.author));

Output:

// D. Form
// R. Martin
// K. Simpson




const events = [
    { title: 'JSConf', venue: 'Convention Center' },
    { title: 'DevOps Days', venue: 'Tech Hub' },
    { title: 'React Summit', venue: 'Online' },
    { title: 'PyCon', venue: 'City Auditorium' }
];



// 1 Mistake - copy-paste

...


// 2 Mistake


...



// 3 Mistake


...








Expected code

events.forEach(event => console.log(event.venue));








Recommendation:

Copying your own code almost always
triggers unexpected errors (forgot to change id or click handle).
You might struggle debugging this error longer,
than you would spend to retype the necessary part of code.

My advice - always type your code, don't copy.










- If you need same functionality again,
think of creating a method for this purpose.

Otherwise, this code is hard to maintain.






Example

We print .title



const books = [
    { title: 'JavaScript', author: 'D. Ford' },
    { title: 'Clean Code', author: 'R. Martin' },
    { title: 'You Do not Know JS', author: 'K. Simpson' }
];

books.forEach(book => console.log(book.title));

const events = [
    { title: 'JSConf', venue: 'Convention Center' },
    { title: 'DevOps Days', venue: 'Tech Hub' },
    { title: 'React Summit', venue: 'Online' },
    { title: 'PyCon', venue: 'City Auditorium' }
];

events.forEach(event => console.log(event.title));


const albums = [
    { title: 'Bad', year: '2025' },
    { title: 'Good', year: '2000' }
];

albums.forEach(album => console.log(album.title));








Let's better organize our code:


const books = [
    { title: 'JavaScript', author: 'D. Ford' },
    { title: 'Clean Code', author: 'R. Martin' },
    { title: 'You Do not Know JS', author: 'K. Simpson' }
];

const events = [
    { title: 'JSConf', venue: 'Convention Center' },
    { title: 'DevOps Days', venue: 'Tech Hub' },
    { title: 'React Summit', venue: 'Online' },
    { title: 'PyCon', venue: 'City Auditorium' }
];

const albums = [
    { title: 'Bad', year: '2025' },
    { title: 'Good', year: '2000' }
];

books.forEach(book => console.log(book.title));
events.forEach(event => console.log(event.title));
albums.forEach(album => console.log(album.title));









Business people ask you to change your code:

log -> warn

// demo in devtools (.log, .warn, .error)


With current code you change 3 times.








A better approach was to create method:

function printTitle(iterable) {
    iterable.forEach(item => console.log(item.title));
}






Use that method in your code:

printTitle(books);
printTitle(events);
printTitle(albums);













Recommendation:

If you repeat yourself 3+ times, think of
creating a method. It will be easier to maintain.
This is a good practice.
